COURSE 01
C# Programming
Structure of a C# Program

Every C program has a defined structure. The entry point is the main() function. Preprocessor directives like #include <stdio.h> are placed at the top to include standard libraries.

Basic structure: Preprocessor directives → Global declarations → main() function → User-defined functions.

The printf() function outputs to the screen. scanf() reads user input. Both require the stdio.h header.

Data Types & Variables

int — whole numbers. float — decimal numbers (6–7 digits precision). double — more precise decimals (15 digits). char — a single character. void — no value.

Variable declaration syntax: datatype variableName = value;. Constants use const keyword or #define directive. Variable names are case-sensitive and cannot start with a number.

Pointers

A pointer is a variable that stores the memory address of another variable. Declared using *: e.g. int *p;. The & operator gets the address of a variable.

Key rule: *p gives the value at the address stored in p (dereferencing). &x gives the address of variable x. Pointers enable dynamic memory allocation using malloc() and free().

COURSE 02
Web Designing
The Box Model

Every HTML element is a rectangular box. The CSS Box Model defines how space is calculated: Content → Padding → Border → Margin.

Use box-sizing: border-box so padding and border are included in the element's total width/height. This is the most predictable behaviour and is standard practice.

Flexbox Essentials

Flexbox is a one-dimensional layout system. Set display: flex on the parent. Key properties: justify-content (main axis alignment), align-items (cross axis alignment), gap (space between items).

Common values: flex-direction: row | column, flex-wrap: wrap, justify-content: center | space-between | space-around.

Responsive Design

Responsive design ensures a website works well on all screen sizes. Key techniques: media queries (@media (max-width: 768px)), fluid widths using percentages or vw/vh, and flexible images with max-width: 100%.

Mobile-first approach: Write default CSS for small screens, then add media queries for larger screens using min-width.

COURSE 03
Database Programming
SQL Basics

SQL (Structured Query Language) is used to communicate with relational databases. It is divided into: DDL (Data Definition Language) — CREATE, ALTER, DROP; DML (Data Manipulation Language) — SELECT, INSERT, UPDATE, DELETE.

Basic query: SELECT column FROM table WHERE condition;. Always end SQL statements with a semicolon.

Joins

Joins combine rows from two or more tables based on a related column. INNER JOIN returns rows where there is a match in both tables. LEFT JOIN returns all rows from the left table plus matched rows from the right. RIGHT JOIN is the reverse.

Syntax: SELECT a.col, b.col FROM tableA a INNER JOIN tableB b ON a.id = b.id;

Normalisation

Normalisation is the process of organising a database to reduce data redundancy and improve integrity.

1NF: Each column contains atomic (indivisible) values; no repeating groups. 2NF: In 1NF + all non-key columns fully depend on the primary key. 3NF: In 2NF + no transitive dependencies (non-key column depending on another non-key column).

COURSE 04
Networking
The OSI Model

The OSI (Open Systems Interconnection) model has 7 layers — a framework for understanding how data travels across a network.

Layer 7 — Application: HTTP, FTP (what users interact with). Layer 4 — Transport: TCP/UDP, end-to-end delivery. Layer 3 — Network: IP addressing and routing. Layer 2 — Data Link: MAC addresses. Layer 1 — Physical: cables and signals.

Memory aid: "All People Seem To Need Data Processing" (Application → Physical).

IP Addressing

An IP address uniquely identifies a device on a network. IPv4 uses 32 bits written as four octets: e.g. 192.168.1.1. Each octet ranges from 0–255.

Classes: Class A (1–126), Class B (128–191), Class C (192–223). Private IP ranges (not routable on the internet): 10.x.x.x, 172.16–31.x.x, 192.168.x.x.

COURSE 05
Systems Analysis
The SDLC

The Software Development Life Cycle (SDLC) is a structured process for building software. Phases: Planning → Analysis → Design → Implementation → Testing → Deployment → Maintenance.

Common models include Waterfall (sequential, each phase completes before the next), Agile (iterative, work in short sprints), and Spiral (risk-driven, iterative).

Data Flow Diagrams (DFD)

A DFD shows how data moves through a system. Key symbols: Process (circle/rectangle — transforms data), Data Store (open-ended rectangle — stores data), External Entity (square — outside the system), Data Flow (arrow — shows direction of data).

Level 0 (Context Diagram): Shows the entire system as one process. Level 1: Breaks down the main process into sub-processes.
